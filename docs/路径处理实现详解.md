# 扫描目录路径处理实现详解

## 概述

XMMusic 使用了一套完整的路径处理系统，用于跨平台（Windows/macOS/Linux）统一管理音乐文件的路径。核心思想是：
- **存储**：路径统一规范化后存储在数据库中
- **查询**：通过 `dir_id` + `file_name` 关联，避免直接存储完整路径
- **重建**：需要完整路径时，通过 `dir_id` 查询 `music_dir` 表获取目录路径，再拼接文件名

## 核心数据结构

### 1. `music_dir` 表（目录表）

```sql
CREATE TABLE music_dir (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  path TEXT UNIQUE NOT NULL,  -- 规范化后的目录路径（如：C:/Music/Rock）
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**作用**：存储所有包含音乐文件的目录路径，每个目录对应一个唯一的 `dir_id`。

### 2. `all_music` 表（音乐表）

```sql
CREATE TABLE all_music (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  dir_id INTEGER NOT NULL,     -- 外键 -> music_dir.id
  file_name TEXT NOT NULL,      -- 文件名（不含路径，如：song.mp3）
  ...
  FOREIGN KEY (dir_id) REFERENCES music_dir(id) ON DELETE RESTRICT,
  UNIQUE(dir_id, file_name)     -- 同一目录下文件名唯一
);
```

**作用**：通过 `dir_id` + `file_name` 关联，而不是直接存储完整路径。

## 路径处理函数详解

### 1. `normalizePath()` - 路径规范化

**位置**：`src/main/database/pathUtils.ts`

**功能**：将不同平台的路径统一规范化

**规则**：
- **Windows**：将反斜杠 `\` 统一转换为正斜杠 `/`，去除末尾分隔符
- **macOS/Linux**：去除末尾分隔符
- **所有平台**：处理多个连续分隔符，合并为单个 `/`

**示例**：
```typescript
normalizePath('C:\\Music\\Rock\\', 'win32')     // => 'C:/Music/Rock'
normalizePath('/Users/name/Music/', 'darwin')  // => '/Users/name/Music'
normalizePath('/home/user/music//', 'linux')   // => '/home/user/music'
```

**为什么需要规范化**：
- 统一存储格式，避免同一目录因路径格式不同而重复存储
- 便于路径比较和查询（如 `LIKE` 查询子目录）

### 2. `parsePath()` - 路径解析

**位置**：`src/main/database/pathUtils.ts`

**功能**：从完整文件路径中分离出目录路径和文件名

**实现**：
```typescript
export function parsePath(
  fullPath: string,
  platform: NodeJS.Platform = process.platform
): { dirPath: string; fileName: string } {
  // 1. 先规范化路径
  const normalizedPath = normalizePath(fullPath, platform)

  // 2. 找到最后一个分隔符
  const lastSeparatorIndex = normalizedPath.lastIndexOf('/')

  // 3. 分离目录和文件名
  const dirPath = normalizedPath.substring(0, lastSeparatorIndex)
  const fileName = normalizedPath.substring(lastSeparatorIndex + 1)

  return {
    dirPath: normalizePath(dirPath, platform),  // 再次规范化确保一致性
    fileName
  }
}
```

**示例**：
```typescript
parsePath('C:/Music/Rock/song.mp3', 'win32')
// => { dirPath: 'C:/Music/Rock', fileName: 'song.mp3' }

parsePath('/Users/name/Music/album/track.flac', 'darwin')
// => { dirPath: '/Users/name/Music/album', fileName: 'track.flac' }
```

### 3. `batchGetOrCreateMusicDir()` - 批量获取/创建目录ID

**位置**：`src/main/database/pathUtils.ts`

**功能**：批量获取或创建目录记录，返回路径到目录ID的映射

**实现流程**：
1. **规范化所有路径**：确保路径格式统一
2. **去重**：使用 `Set` 去除重复路径
3. **批量查询已存在的目录**：使用 `IN` 子句一次性查询
4. **批量插入新目录**：使用事务批量插入不存在的目录
5. **返回映射**：`{ [path]: dirId }`

**性能优化**：
- 使用事务批量插入，减少数据库往返
- 先查询后插入，避免重复插入

**示例**：
```typescript
const dirMap = batchGetOrCreateMusicDir(db, [
  'C:/Music/Rock',
  'C:/Music/Jazz',
  'C:/Music/Classical'
], 'win32')

// => {
//   'C:/Music/Rock': 1,
//   'C:/Music/Jazz': 2,
//   'C:/Music/Classical': 3
// }
```

### 4. `buildPathFromMusicRecord()` - 从数据库记录构建完整路径

**位置**：`src/main/database/pathUtils.ts`

**功能**：通过 `dir_id` 和 `file_name` 重建完整文件路径

**实现流程**：
1. 通过 `dir_id` 查询 `music_dir` 表获取目录路径
2. 使用 `buildFullPath()` 拼接目录路径和文件名
3. 根据平台使用正确的路径分隔符（Windows 用 `\`，其他用 `/`）

**示例**：
```typescript
// 数据库记录：{ dir_id: 1, file_name: 'song.mp3' }
// music_dir 表：{ id: 1, path: 'C:/Music/Rock' }
const fullPath = buildPathFromMusicRecord(db, { dir_id: 1, file_name: 'song.mp3' }, 'win32')
// => 'C:\Music\Rock\song.mp3'  (Windows 使用反斜杠)
```

### 5. `buildFullPath()` - 构建完整路径

**位置**：`src/main/database/pathUtils.ts`

**功能**：将目录路径和文件名拼接成完整路径

**实现**：
```typescript
export function buildFullPath(
  dirPath: string,
  fileName: string,
  platform: NodeJS.Platform = process.platform
): string {
  const normalizedDir = normalizePath(dirPath, platform)
  const separator = platform === 'win32' ? '\\' : '/'
  return `${normalizedDir}${separator}${fileName}`
}
```

**注意**：
- 存储时统一使用正斜杠（规范化）
- 使用时根据平台转换为正确的分隔符

## 扫描流程中的路径处理

### 扫描单个目录流程

**位置**：`src/main/services/fileScanner.ts` - `scanDirectory()`

#### 步骤 1：规范化根目录路径

```typescript
const normalizedRoot = normalizePath(path, process.platform)
// 例如：'C:\\Music\\Rock' => 'C:/Music/Rock'
```

#### 步骤 2：确定需要同步的目录ID列表

**递归扫描**（`options.recursive = true`）：
```typescript
// 查询根目录及其所有子目录
const rootLike = `${normalizedRoot}/%`
scannedDirIds = sqlite.prepare(`
  SELECT id FROM music_dir
  WHERE path = ? OR path LIKE ?
`).all(normalizedRoot, rootLike).map(r => r.id)
```

**非递归扫描**（`options.recursive = false`）：
```typescript
// 仅获取根目录本身的 dir_id
const map = batchGetOrCreateMusicDir(sqlite, [normalizedRoot], process.platform)
rootDirId = map[normalizedRoot]
scannedDirIds = [rootDirId]
```

#### 步骤 3：收集所有音乐文件

```typescript
const files = await this.collectFiles(normalizedRoot, options)
// files = [
//   'C:/Music/Rock/song1.mp3',
//   'C:/Music/Rock/album/track1.flac',
//   ...
// ]
```

#### 步骤 4：批量获取目录ID映射

```typescript
// 从所有文件中提取目录路径
const dirPaths = [...new Set(files.map(f => parsePath(f, process.platform).dirPath))]
// dirPaths = ['C:/Music/Rock', 'C:/Music/Rock/album', ...]

// 批量获取或创建目录ID
const dirIdMap = batchGetOrCreateMusicDir(this.db.getDatabase(), dirPaths, process.platform)
// dirIdMap = {
//   'C:/Music/Rock': 1,
//   'C:/Music/Rock/album': 2,
//   ...
// }
```

#### 步骤 5：构建文件名集合（用于存在性同步）

```typescript
const foundFileNamesByDirId = new Map<number, Set<string>>()
for (const file of files) {
  const { dirPath, fileName } = parsePath(file, process.platform)
  const normalizedDirPath = normalizePath(dirPath, process.platform)
  const dirId = dirIdMap[normalizedDirPath]

  if (!dirId) continue

  let set = foundFileNamesByDirId.get(dirId)
  if (!set) {
    set = new Set<string>()
    foundFileNamesByDirId.set(dirId, set)
  }
  set.add(fileName)
}
// foundFileNamesByDirId = {
//   1: Set(['song1.mp3', 'song2.mp3']),
//   2: Set(['track1.flac']),
//   ...
// }
```

#### 步骤 6：处理每个文件

```typescript
for (const file of files) {
  // 1. 解析路径
  const { dirPath, fileName } = parsePath(file, process.platform)
  const normalizedDirPath = normalizePath(dirPath, process.platform)

  // 2. 获取目录ID
  const dirId = dirIdMap[normalizedDirPath]

  // 3. 检查文件是否已存在
  const existing = db.prepare(`
    SELECT id FROM all_music
    WHERE dir_id = ? AND file_name = ?
  `).get(dirId, fileName)

  // 4. 插入或更新记录
  if (!existing) {
    const musicId = db.insertAllMusic({
      dir_id: dirId,
      file_name: fileName,
      ...
    })
  }
}
```

#### 步骤 7：同步文件存在性状态

**非递归扫描**（扫描开始前）：
```typescript
// 先把该目录下所有记录标记为不存在
sqlite.prepare(`
  UPDATE all_music
  SET is_exists = 0, updated_at = CURRENT_TIMESTAMP
  WHERE dir_id = ? AND is_exists != 0
`).run(rootDirId)
```

**扫描结束后**：
```typescript
// 把本次扫描到的文件置回存在
for (const dirId of scannedDirIds) {
  const found = foundFileNamesByDirId.get(dirId)
  if (found && found.size > 0) {
    const names = Array.from(found)
    // 分批更新（避免 SQLite 参数限制）
    for (let i = 0; i < names.length; i += CHUNK_SIZE) {
      const chunk = names.slice(i, i + CHUNK_SIZE)
      sqlite.prepare(`
        UPDATE all_music
        SET is_exists = 1, updated_at = CURRENT_TIMESTAMP
        WHERE dir_id = ? AND file_name IN (${placeholders})
      `).run(dirId, ...chunk)
    }
  }
}
```

## 路径处理的优势

### 1. 跨平台兼容性
- 统一使用规范化路径存储，避免平台差异
- 使用时根据平台转换分隔符

### 2. 存储效率
- 不重复存储相同目录路径（通过 `dir_id` 关联）
- 同一目录下的多个文件共享一个 `dir_id`

### 3. 查询性能
- 通过 `dir_id` 索引快速查询
- 支持按目录路径模式查询（`LIKE` 查询子目录）

### 4. 数据一致性
- 路径规范化确保同一目录不会因格式不同而重复
- `UNIQUE(dir_id, file_name)` 约束防止重复记录

### 5. 灵活性
- 可以轻松查询某个目录下的所有文件
- 可以查询某个根目录及其所有子目录的文件

## 路径处理流程图

```
用户输入路径
    ↓
normalizePath() 规范化
    ↓
collectFiles() 收集文件
    ↓
parsePath() 解析每个文件路径
    ↓
batchGetOrCreateMusicDir() 批量获取/创建目录ID
    ↓
存储到 all_music (dir_id + file_name)
    ↓
查询时 buildPathFromMusicRecord() 重建完整路径
    ↓
返回给前端使用
```

## 注意事项

1. **路径规范化必须在所有路径操作前进行**，确保一致性
2. **Windows 路径处理**：存储时用正斜杠，使用时用反斜杠
3. **批量操作**：使用 `batchGetOrCreateMusicDir()` 而不是逐个调用，提高性能
4. **事务处理**：批量插入目录时使用事务，确保原子性
5. **参数限制**：SQLite 有 999 个绑定参数限制，需要分批处理大量文件

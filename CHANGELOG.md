# 变更记录

所有重要的项目变更都将记录在此文件中。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，
本项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [1.0.3] - 2025-11-24

### 新增
- **文件名列显示** - 在所有歌曲列表中新增文件名列
  - 本地音乐、我喜欢、最近播放、我的歌单等所有列表都支持
  - 文件名列位于专辑和时长列之间，宽度 calc(20% + 20px)
  - 支持文本溢出显示省略号，悬停显示完整文件名
  - 便于用户识别和管理文件

- **智能标签编辑器** - 修复乱码ID3标签的利器
  - 右键菜单新增"编辑标签"选项（带编辑图标）
  - 对话框形式的标签编辑界面，简洁易用
  - 智能解析文件名自动填充标签信息
  - 支持多种文件命名格式识别:
    - `歌手-歌曲.mp3`
    - `歌手&歌手-歌曲.mp3` (多歌手)
    - `歌手_歌曲.mp3` (下划线分隔)
    - `歌曲名.mp3` (仅歌曲名)
  - **对调按钮** - 一键交换歌手和歌曲名，轻松处理"歌曲名-歌手名"格式
  - 现代化UI设计，带加载状态和错误提示

### 修复
- **修复启动时播放状态恢复问题**
  - 移除不可靠的自动播放逻辑，改为只恢复播放状态
  - 智能检测音频实例，首次播放时自动重新加载音乐
  - 修复启动时歌曲时长显示为0的问题
  - 自动恢复到上次保存的播放位置
  - 符合主流音乐播放器的用户体验设计
- 修复GitHub Actions构建失败问题，添加缺失的vuedraggable依赖
- 修复主题类型检查，避免类型错误

### 改进
- **元数据更新全局同步** - 标签编辑保存后，所有视图实时更新
  - 发现音乐、本地音乐、我喜欢、最近播放全部同步
  - 我的歌单详情页实时更新
  - 正在播放的歌曲信息实时刷新
  - 播放队列中的歌曲信息同步更新
  - 采用精准更新策略，只更新被修改的歌曲，性能更优
- 优化歌曲列表列宽分配，提升信息展示效率
- 统一所有播放列表组件的显示格式
- 改进元数据编辑用户体验
## [1.0.2] - 2025-11-24

### 重大变更
- **UI全面重构** - 采用仿QQ音乐设计风格
  - 全新的现代化界面设计,视觉体验大幅提升
  - 重新设计播放器布局,更符合用户使用习惯
  - 优化颜色方案和视觉层次
  - 改进交互反馈和动画效果

### 新增
- 使用 Lucide Icons 替换所有 emoji 图标
  - 界面图标更统一、更现代
  - 包括路由图标、功能按钮、状态指示器等
  - 提升整体视觉一致性

### 修复
- 修复窗口切换bug和性能问题
  - 修复正常窗口和Mini窗口反复切换时窗口尺寸逐渐变小的问题
  - 修复Mini窗口切换时GPU错误导致的严重卡顿问题
  - 正确重置 `normalBounds` 变量,确保窗口尺寸准确保存和恢复
- 修复macOS窗口红绿灯失焦颜色问题
  - 实现窗口外观(nativeTheme)跟随应用主题而非系统主题
  - 确保所有主题组合下红绿灯失焦时都显示正确的灰色
  - 解决系统深色+应用浅色时红绿灯失焦显示白色(看不清)的问题

### 性能优化
- 添加 keep-alive 组件缓存,避免 Mini 窗口重复创建/销毁
- 移除 Mini 窗口背景模糊效果,使用简洁的渐变背景
  - 彻底消除 GPU 资源泄漏和 "Invalid mailbox" 错误
  - 切换流畅,无卡顿
- 优化 header 按钮 hover 效果,减少响应延迟(200ms → 150ms)

### 改进
- Mini 窗口还原按钮添加 hover 背景高亮效果
- 优化整体UI交互体验
- 提升应用视觉质量和使用流畅度

### 技术优化
- 实现窗口状态管理优化
- 添加 nativeTheme API 支持
- 改进组件生命周期管理

## [1.0.1] - 2025-11-20

### 新增
- 开发环境和生产环境数据库分离（开发环境使用 `xmmusic-dev.db`）
- 播放队列状态图标，实时显示歌曲是否在播放队列中
- 关于页面添加详细的技术栈说明

### 修复
- 修复均衡器功能不工作的问题
  - 将 audio 元素添加到 DOM，使 Web Audio API 能够连接
  - 改进均衡器初始化和缓存更新逻辑
- 修复均衡器页面切换时崩溃的问题（添加数据验证）
- 修复歌单功能缓存更新问题
- 静默处理数据库关闭时的 SQLITE_BUSY 错误（不影响数据完整性）
- 修复播放历史页面双击播放时的崩溃问题
- 修复本地图片加载受 Electron 安全策略限制的问题（添加 `local-file://` 协议）
- 修复 Windows 环境下主题切换时月亮和太阳图标同时显示的bug（Header.vue 的 v-else 指令语法错误）
- 修复 Footer 中歌曲名称宽度随内容变化的问题（设置固定宽度200px并添加文本溢出省略号）
- 修复播放控制按钮（快进、快退、播放、暂停）图标过小的问题（从40px增大到48px）
- 修复中文文件名的 MP3 播放失败问题（移除不必要的 encodeURIComponent 编码）

### 移除
- 移除"播放统计"和"智能推荐"功能（包括菜单项、组件和相关代码），精简应用体积
- 移除相似歌曲推荐功能

### 新增
- 音乐封面提取和显示功能
  - 扫描时自动从音乐文件提取封面并保存
  - 播放器左下角显示当前播放音乐的封面
  - 支持 JPG 和 PNG 格式
  - 自动更新缺少封面的现有记录

### 改进
- 统一术语：将"播放列表"改为"歌单"（Playlist），"播放队列"（Queue）概念更清晰
- 移除播放队列操作成功时的弹窗提示，图标状态变化已足够提示用户
- 优化用户体验，减少不必要的提示弹窗

### 技术栈更新
- 更新关于页面显示的技术栈版本信息
  - Electron 39.2.1
  - Vue 3.5.24
  - TypeScript 5.9.3
  - Vite 7.2.2
  - Pinia 3.0.4
  - Vue Router 4.4.5
  - Howler.js 2.2.4
  - Lucide Icons 0.554.0

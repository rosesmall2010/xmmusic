# 变更记录

所有重要的项目变更都将记录在此文件中。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，
本项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [1.0.8] - 2025-12-12

### 修复
- **编辑标签保存失败**
  - 修复编辑标签保存时报 `no such table: music` 的错误
  - 使用 `updateAllMusic` 方法更新 `all_music` 表，替代旧的 `music` 表
  - 修正字段名映射：`coverPath` -> `cover_path`
- **标签修改后显示区域不同步**
  - 修复标签修改保存后，各显示区域（本地音乐列表、最近播放、歌单详情、收藏列表、播放队列等）没有同步更新的问题
  - 统一所有视图的元数据更新逻辑，使用重新赋值整个数组的方式触发响应式更新
  - 修复 playerStore 中的队列和当前音乐的同步更新逻辑
- **收藏图标状态切换延迟**
  - `toggle-favorite` IPC 直接返回最新收藏状态，减少一次 IPC 往返
  - 列表收藏点击改为先本地立即切换，再异步落库并回填，提升空心/实心切换速度
- **播放队列图标状态切换延迟**
  - 播放队列图标状态判断统一以响应式队列 Set 为准，避免 `inQueue` 字段在 shallowRef 列表中变更不触发渲染
  - 修复批量添加到播放队列时的状态判断逻辑，确保图标及时更新
- **随机播放下一首可能重复当前歌曲**
  - 随机模式下当队列歌曲数量大于 1 时，下一首将不会重复刚刚播放的那一首

### 改进
- **正在播放界面优化**
  - 音效设置对话框改为在整个app居中显示（使用fixed定位）
  - 音量控制与播放按钮垂直居中对齐
  - 移除音量百分比显示，简化界面
  - 优化音量进度条和音量图标的对齐
- **正在播放界面背景特效**
  - 新增火焰背景特效，并根据歌曲封面主色与播放频谱动态变化
- **正在播放/迷你模式播放控制栏统一**
  - 播放/暂停按钮同步为胶囊形状，并统一主色、阴影与 hover 交互
  - 调整控制区域高度与间距，使上一首/下一首等按钮与主按钮对齐
- **主界面播放控制栏优化**
  - 播放/暂停按钮从圆形改为胶囊形状（中间矩形，左右两个半圆）
  - 其他控制按钮（快进、快退、播放模式、播放队列）hover时只显示图标放大和亮色效果，移除圆形背景
  - 默认封面图标填充满封面框（添加 `mode="fill"` 属性）
  - 喜欢按钮从左侧移到右侧音效控制按钮的左边
  - 调整播放进度条位置，使其适当上移并优化与播放控制区域的间隔
- **主界面顶部优化**
  - 搜索框从圆形改为矩形圆角框
  - 在顶部空白区域添加可拖动区域，方便移动窗口
  - 修复搜索框两边空白区域无法拖动的问题

## [1.0.7] - 2025-12-12

### 新增
- **不可播放文件标记**
  - Howler 加载失败时自动写入 `all_music.is_playable=0`、`play_error_reason`、`is_corrupted=1`
  - 新增 `update-music-play-status` IPC，供渲染层上报播放失败原因
- **列表状态图标**
  - 歌曲列表在标题旁显示文件状态图标：文件不存在 / 不可播放（悬停显示原因）
- **扫描缺失文件标记**
  - 扫描结束后按目录对比“本次扫描文件列表”与 `all_music`，自动标记缺失文件 `is_exists=0`
  - 非递归扫描（不含子目录）支持“先全置不存在，再把实际存在的置回存在”

### 修复
- **右键菜单边缘遮挡**
  - 修复在屏幕最右侧/底部右键时菜单被裁切的问题，自动调整到可视区域
- **列表图标状态不同步**
  - 修复“我喜欢/播放队列”图标需要滚动或切换窗口后才刷新显示的问题
- **歌词自动匹配失败**
  - 改进同目录歌词自动查找（文件名提取更可靠，增强日志与错误处理）
  - 修复歌词路径回写数据库逻辑，确保下次可直接读取
- **音效设置保存失败**
  - 修复均衡器设置通过 IPC 保存时报 `An object could not be cloned`（将响应式数据转换为纯 JSON 后再保存）
- **音量保存/恢复异常**
  - PlayerBar 音量改为直接绑定 `playerStore.volume`，并同步到播放器 `setVolume`，确保启动恢复与保存一致

### 文档
- 新增《扫描目录路径处理实现详解》文档，说明路径规范化、目录ID映射与扫描链路

### 变更
- **数据库驱动升级**
  - 将 `@vscode/sqlite3` + `deasync` 替换为 `better-sqlite3`
  - 性能提升：better-sqlite3 相比原方案快 3-20 倍（视操作类型而定）
  - 简化架构：移除 deasync 同步 hack，使用 better-sqlite3 原生同步 API
  - sqlite3-sync.ts 包装层从 218 行精简至 ~80 行
